🔗 B - tcaF (abc389 B)

tags: 模擬(Simulation) 二分搜尋(Binary Search)

題意

給定一個不小於 222 的整數 XXX，找出正整數 NNN 使得 N!=XN! = XN!=X。

約束條件

  • 2≤X≤3×10182 \leq X \leq 3 \times 10^{18}2≤X≤3×1018

思路：模擬(Simulation) / 二分搜尋(Binary Search)

由於只有一個詢問，因此可以從 111 開始枚舉 NNN，直到 N!=XN! = XN!=X 為止。

如果有多個詢問的話，可以考慮預處理後使用二分搜尋。

🔗 C - Snake Queue (abc389 C)

tags: 佇列(Queue) 模擬(Simulation)

題意

原本的題目敘述是以下問題的數學化：

有一群蛇在排隊，而每條蛇會佔據一個座標區間 [x,x+l][x, x + l][x,x+l]，其中 xxx 為蛇的頭部座標，lll 為蛇的長度，且隊伍中第一條蛇的頭部座標為 000。

現在有 QQQ 個操作，而每個操作有以下三種：

  • 操作 111：1 l。將一條長度為 lll 的蛇加入到隊伍的末尾，注意其必須緊跟在最後一條蛇的尾巴後面。
  • 操作 222：2。隊伍最前面的蛇離開隊伍，則後續的蛇往前移動，直到隊伍中第一條蛇的頭部座標為 000 為止。
  • 操作 333：3 k。輸出隊伍中從前面數第 kkk 條蛇的頭部座標。

    Snake1    : 0, 2
    Snake2    : 2, 3
    Snake3    : 3, 6
    Snake4    : 6, 10

約束條件

  • 1≤Q≤3×1051 \leq Q \leq 3 \times 10^{5}1≤Q≤3×105
  • 對於操作 111，1≤l≤1091 \leq l \leq 10^{9}1≤l≤109
  • 對於操作 222，保證佇列不為空。
  • 對於操作 333，令 nnn 為佇列中的蛇的數量，則 1≤k≤n1 \leq k \leq n1≤k≤n。

思路：模擬(Simulation)

由於題目已經給出清晰的操作定義，且題目名稱也已經暗示了這是一道佇列問題，我們可以使用佇列（Queue）來處理這些操作，維護隊伍中的蛇的頭部座標即可。

然而，特別需要注意的是，當執行第二種操作（移除最前面的蛇）時，後續的蛇會因為需要向左平移而改變其座標。但顯然不能真正的將所有蛇的頭部座標減少。

為此可以維護一個變數 sss，代表所有蛇的總平移量，每當執行移除操作時，將最前面的蛇的長度加到這個變數 sss 中。對於查詢操作，我們僅需借助這個變數 sss 去計算實際的頭部座標即可。如此我們便可以保存未平移的頭部座標，只要在輸出時減去 sss 即可。

例如以下狀況，此時 s=3s = 3s=3，因此第 222 條蛇的頭部座標為 6−s=36 - s = 36−s=3。

    dateFormat x
    axisFormat %Q
    
    Snake3    : 3, 6
    Snake4    : 6, 10

複雜度分析

  • 時間複雜度：O(Q)\mathcal{O}(Q)O(Q)
  • 空間複雜度：O(Q)\mathcal{O}(Q)O(Q)

🔗 D - Squares in Circle (abc389 D)

tags: 幾何(Geometry) 雙指標(Two Pointers)

題意

在二維座標平面上，存在一個無限延伸的 1×11 \times 11×1 正方形棋盤。

考慮在這些正方形的中心畫一個半徑為 RRR 的圓。這些正方形中有多少個完全包含在圓內？

更準確地說，找出符合條件的整數對 (i,j)(i,j)(i,j) 的數量，使得所有四個點 (i+0.5,j+0.5)(i+0.5,j+0.5)(i+0.5,j+0.5)、(i+0.5,j−0.5)(i+0.5,j-0.5)(i+0.5,j−0.5)、(i−0.5,j+0.5)(i-0.5,j+0.5)(i−0.5,j+0.5) 和 (i−0.5,j−0.5)(i-0.5,j-0.5)(i−0.5,j−0.5) 與原點的距離均不超過 RRR。

約束條件

  • 1≤R≤1061 \leq R \leq 10^{6}1≤R≤106

思路：幾何(Geometry) + 雙指標(Two Pointers)

根據上圖，可以將問題分為以下 3 種情況：

  1. 正方形中心在圓心，由於 R≥1R \geq 1R≥1，因此固定是 111 個
  2. 正方形中心在 XXX 軸或 YYY 軸上
    • 在正 XXX 軸上有 R−1R - 1R−1 個，因此總共有 4×(R−1)4 \times (R - 1)4×(R−1) 個
  3. 正方形中心在象限內
    • 由於在二維平面上，圓與坐標軸具有 對稱性。也就是說，若一個正方形位於第一象限內符合條件，則其在其他三個象限的對應位置的正方形也同樣符合條件。因此，我們只需計算第一象限內的符合條件的正方形數量，並將結果乘以 444，即可得到所有象限內的總數。
    • 在第一象限中，判斷中心點座標為 (i,j)(i, j)(i,j) 的正方形是否在圓內，等同於檢查 (i+0.5,j+0.5)(i + 0.5, j + 0.5)(i+0.5,j+0.5) 到圓心的距離是否 ≤R\leq R≤R
    • 由於在第一象限中，當 iii 增加時， jjj 只可能不變或減少，因此可以使用雙指標，從 i=1,j=Ri = 1, j = Ri=1,j=R 開始，枚舉 iii 並維護最大的 jjj

複雜度分析

  • 時間複雜度：O(R)\mathcal{O}(R)O(R)
    • 枚舉 iii 需要 O(R)\mathcal{O}(R)O(R)，而雖然內層迴圈看似也需要 O(R)\mathcal{O}(R)O(R)，但由於 jjj 只會減少，因此 iii 和 jjj 實際上都只會最多減少 O(R)\mathcal{O}(R)O(R) 次。
  • 空間複雜度：O(1)\mathcal{O}(1)O(1)

🔗 E - Square Price (abc389 E)

tags: 二分搜尋(Binary Search)

題意

有 NNN 種產品，每種產品的庫存量均為 1010010^{100}10100 單位。

你可以購買每種產品的任意非負數量的單位。若要購買 kkk 單位的第 iii 種產品，則需要花費 k2×Pik^2 \times P_ik2×Pi​ 日元。

如果你的總購買成本最多為 MMM 日元，那麼你最多可以購買多少單位的產品總數？

約束條件

  • 1≤N≤2×1051 \leq N \leq 2 \times 10^{5}1≤N≤2×105
  • 1≤M≤10181 \leq M \leq 10^{18}1≤M≤1018
  • 1≤Pi≤2×1091 \leq P_i \leq 2 \times 10^{9}1≤Pi​≤2×109

思路

由於購買 kkk 件商品的總花費為 p⋅k2p \cdot k^2p⋅k2，故可以計算出購買第 jjj 件商品的價格為 p⋅(j2−(j−1)2)=p⋅(2×j−1)p \cdot (j^2 - (j-1)^2) = p \cdot (2 \times j - 1)p⋅(j2−(j−1)2)=p⋅(2×j−1)，故我們可以知道所有 N×10100N \times 10^{100}N×10100 件商品的價格。但顯然不可能列舉出所有商品的價格，而維護一個大小為 NNN 的 Heap 也是不可行的，推導如下：

  • 若所有商品的價格均為 111，且每種商品均購買 kkk 件，則總價格為 N⋅k2≤MN \cdot k^2 \leq MN⋅k2≤M，得 k≤MNk \leq \sqrt{\frac{M}{N}}k≤NM​​
  • 而我們需要對 Heap 操作 k⋅Nk \cdot Nk⋅N 次，因此需要 O(k⋅Nlog⁡N)=O(M⋅Nlog⁡N)\mathcal{O}(k \cdot N \log N) = \mathcal{O}(\sqrt{M \cdot N} \log N)O(k⋅NlogN)=O(M⋅N​logN) 的時間複雜度，顯然不可行。

因此考慮 二分搜尋(Binary Search) ，找到一個整數 xxx，使得我們可以 購買所有價格不超過 x 的商品。對於價格為 ppp 的商品，若購買 kkk 件，則需滿足 p⋅(2k−1)≤xp \cdot (2k-1) \leq xp⋅(2k−1)≤x ，此時總價格為 p⋅k2p \cdot k^2p⋅k2。

二分找到最大的 xxx 後，由於此時可能還可以購買部分價格為 x+1x+1x+1 的商品，因此先用 x+1x+1x+1 計算總價格，若總價格超過 MMM，則代表超過部分的商品價格都是 x+1x+1x+1，此時將超過的商品數量減去，即可得到答案。

複雜度分析

  • 時間複雜度：O(Nlog⁡M)\mathcal{O}(N \log M)O(NlogM)
    • 二分搜尋可以購買的商品最大價格 xxx 需要 O(log⁡M)\mathcal{O}(\log M)O(logM) 的時間
    • 檢查函數需要遍歷所有商品，因此需要 O(N)\mathcal{O}(N)O(N) 的時間
  • 空間複雜度：O(N)\mathcal{O}(N)O(N)

🔗 F - Rated Range (abc389 F)

tags: 線段樹(Segment Tree) 懶標記線段樹(Lazy Segment Tree) 二分搜尋(Binary Search) 動態規劃(Dynamic Programming)

題意

有 NNN 場比賽，在第 iii 場比賽中（1≤i≤N1 \leq i \leq N1≤i≤N），如果參賽選手的 rating 在 LiL_iLi​ 和 RiR_iRi​ 之間（包含邊界），則他的 rating 增加 111，每個選手都會參加所有比賽。

給定 QQQ 個查詢，每個查詢提供一個整數 XXX，表示某選手的初始 rating，請計算選手參加完所有比賽後的最終 rating。

約束條件

  • 1≤N≤2×1051 \leq N \leq 2 \times 10^51≤N≤2×105
  • 1≤Li≤Ri≤5×1051 \leq L_i \leq R_i \leq 5 \times 10^51≤Li​≤Ri​≤5×105 (1≤i≤N)(1 \leq i \leq N)(1≤i≤N)
  • 1≤Q≤3×1051 \leq Q \leq 3 \times 10^51≤Q≤3×105
  • 1≤X≤5×1051 \leq X \leq 5 \times 10^51≤X≤5×105

思路：線段樹(Segment Tree) + 二分搜尋(Binary Search)

首先考慮最暴力的情況，對於每一個詢問，我們都模擬一遍該選手參加所有比賽的過程，這樣的時間複雜度是 O(NQ)O(NQ)O(NQ) 的，這種複雜度是不可接受的。

但可以注意到，每次更新 rating 時，我們只會對一個區間進行操作，而這樣的操作是可以進行合併的。我們可以令 f[j][i]f[j][i]f[j][i] 表示當初始分數為 jjj 時，參加完第 iii 場比賽後的分數，則有：

f[j][i]={f[j][i−1]+1if Li≤f[j][i−1]≤Rif[j][i−1]otherwisef[j][i] = \begin{cases} f[j][i-1] + 1 & \text{if } L_i \leq f[j][i-1] \leq R_i \\ f[j][i-1] & \text{otherwise} \end{cases} f[j][i]={f[j][i−1]+1f[j][i−1]​if Li​≤f[j][i−1]≤Ri​otherwise​

邊界條件為 f[j][0]=jf[j][0] = jf[j][0]=j，所求為 f[j][n]f[j][n]f[j][n]，其中 j=Xj = Xj=X。

因此可以考慮維護 f[j][i]f[j][i]f[j][i]，由於我們不需要中間的結果，因此只要維護 f[j]f[j]f[j] 即可。那麼當參加完第 iii 場比賽後，我們可以更新所有滿足 Li≤f[j]≤RiL_i \leq f[j] \leq R_iLi​≤f[j]≤Ri​ 的 jjj，使用 稀疏表(Sparse Table) 或 線段樹(Segment Tree) 的話，區間更新可以在 O(log⁡X)O(\log X)O(logX) 的時間內完成，其中 XXX 為 rating 的值域，注意區間更新需要使用 懶標記線段樹(Lazy Segment Tree)。

但這樣還有一個問題，如何找到 f[j][i−1]f[j][i - 1]f[j][i−1] 中的 LiL_iLi​ 和 RiR_iRi​ 的位置呢？這裡可以利用 f[j]f[j]f[j] 的特性，注意到當 i=0i = 0i=0 時，f[j]f[j]f[j] 的值是單調遞增的，而當 i>0i > 0i>0 時，f[j]f[j]f[j] 的值也一定是單調不減的，簡略的證明如下：

  • 注意到每次比賽只會將符合條件的 rating 增加 111，而初始時 f[j][0]=jf[j][0] = jf[j][0]=j 是單調不減的。每次比賽後，滿足條件的 f[j]f[j]f[j] 增加 111，這最多只會讓 f[j]f[j]f[j] 增加到和 f[j+1]f[j+1]f[j+1] 相同，因此 f[j]f[j]f[j] 是單調不減的。

根據這個性質，任何時刻的 f[j]f[j]f[j] 都是單調不減的，因此可以使用 二分搜尋(Binary Search) 來找到最小的 jjj 使得 f[j]≥Lif[j] \geq L_if[j]≥Li​，以及最大的 jjj 使得 f[j]≤Rif[j] \leq R_if[j]≤Ri​。這樣可以確定需要更新的連續區間，從而利用線段樹進行區間更新。

為方便起見，這裡直接使用 AtCoder Library 的 懶標記線段樹(Lazy Segment Tree)。

複雜度分析

  • 時間複雜度：O(X+Nlog⁡X+Qlog⁡X)\mathcal{O}(X + N \log X + Q \log X)O(X+NlogX+QlogX)
    • 初始化線段樹需要 O(X)\mathcal{O}(X)O(X)
    • 對於每場比賽，需要 O(log⁡X)\mathcal{O}(\log X)O(logX) 的時間找到 LiL_iLi​ 和 RiR_iRi​ 的位置
    • 對於每個查詢，需要 O(log⁡X)\mathcal{O}(\log X)O(logX) 的時間找到 f[j][n]f[j][n]f[j][n] 的值
  • 空間複雜度：O(N+X)\mathcal{O}(N + X)O(N+X)
    • 線段樹需要 O(X)\mathcal{O}(X)O(X) 的空間
