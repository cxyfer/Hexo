🔗 CF83B. Doctor

tags: 二分搜尋(Binary Search) 模擬(Simulation)

題意

有 nnn 個編號 111 到 nnn 的對象排成一列，每個對象需要被處理 aia_iai​ 次。

每次操作時，處理隊伍最前面的對象一次：

如果這個對象還沒被處理完（次數未達 aia_iai​），就會回到隊伍最後面。
如果已經處理滿 aia_iai​ 次，就離開隊伍。
總共會進行 kkk 次操作。請輸出第 kkk 次操作結束後，隊伍中剩下的對象編號，按照順序輸出。

如果在第 kkk 次操作之前，所有對象都已經離開隊伍，輸出 -1。

約束條件

  • 1≤n≤1051 \leq n \leq 10^{5}1≤n≤105
  • 0≤k≤10140 \leq k \leq 10^{14}0≤k≤1014
  • 1≤ai≤1091 \leq a_i \leq 10^{9}1≤ai​≤109

思路：二分搜尋

本題的目標是，在操作 kkk 次後，確定隊伍中剩餘對象的順序。由於 kkk 可能高達 101410^{14}1014，若逐次模擬每次看診，效率極低。因此需要優化，可以從總共循環了幾輪來思考。

假設進行了完整的 mmm 輪，也就是每隻對象最多被處理 mmm 次。對於第 iii 個對象，實際處理次數為 min⁡(ai,m)\min(a_i, m)min(ai​,m)，其中 aia_iai​ 是該對象的需求次數。因此，完成 mmm 輪後的總處理次數為：S(m)=∑i=1nmin⁡(ai,m)S(m) = \sum_{i=1}^n \min(a_i, m)S(m)=∑i=1n​min(ai​,m)

我們希望找到最大的 mmm，使得總次數不超過 kkk，即滿足 S(m)≤kS(m) \leq kS(m)≤k，這可以用二分搜尋來找到。

確定此最大 mmm 後，剩餘的操作次數（未完成的處理）最多只會少於一整輪，也就是少於 nnn 次，可以直接模擬即可。

複雜度分析

  • 時間複雜度：O(nlog⁡M)O(n \log M)O(nlogM)
  • 空間複雜度：O(n)O(n)O(n)
