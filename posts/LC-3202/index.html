🔗 🟡 3202. Find the Maximum Length of Valid Subsequence II

tags: 陣列(Array) 動態規劃(Dynamic Programming)

題意

給定一個整數陣列 numsnumsnums 和一個正整數 kkk。

如果 numsnumsnums 的一個長度為 xxx 的 子序列(Subsequence) subsubsub 滿足以下條件，則稱其為有效的：

  • (sub[0]+sub[1])mod  k==(sub[1]+sub[2])mod  k==⋯==(sub[x−2]+sub[x−1])mod  k(sub[0] + sub[1]) \mod k == (sub[1] + sub[2]) \mod k == \cdots == (sub[x - 2] + sub[x - 1]) \mod k(sub[0]+sub[1])modk==(sub[1]+sub[2])modk==⋯==(sub[x−2]+sub[x−1])modk

返回 numsnumsnums 中最長的有效子序列的長度。

約束條件

  • 2≤nums.length≤1032 \leq \text{nums.length} \leq 10^32≤nums.length≤103
  • 1≤nums[i]≤1071 \leq \text{nums[i]} \leq 10^71≤nums[i]≤107
  • 1≤k≤1031 \leq k \leq 10^31≤k≤103

思路：動態規劃(Dynamic Programming)

首先，我們觀察到條件 (a+b)(modk)=s(a + b) \pmod k = s(a+b)(modk)=s 只跟 a(modk)a \pmod ka(modk) 和 b(modk)b \pmod kb(modk) 有關，而與 aaa 和 bbb 的原始數值無關。因此，我們可以先對 numsnumsnums 陣列中的所有數字進行預處理，將它們全部轉換為模 kkk 的餘數。這可以大大簡化後續的計算。

方法一：枚舉子序列中相鄰兩項的和 sss

這道題要求我們找出一個最長的子序列，使得子序列中任意相鄰兩項的和，在模 kkk 意義下都是一個定值。

由於這個定值是多少我們事先並不知道，一個直觀的想法是枚舉所有可能的定值。令這個定值為 sss，其中 sss 的可能範圍是 0,1,…,k−10, 1, \dots, k-10,1,…,k−1。對於每一個固定的 sss，求解滿足 (sub[i-1] + sub[i]) % k == s 的最長子序列長度。最後，取所有 sss 對應的最長長度中的最大值，即為本題的答案。

接下來的問題是，對於一個固定的 sss，如何找出最長的有效子序列？這是一個典型的 動態規劃(Dynamic Programming) 問題。我們可以定義一個大小為 (n+1)×k(n + 1) \times k(n+1)×k 的 DP 陣列 fff，其中 f[i][j]f[i][j]f[i][j] 表示考慮前 iii 個數，任兩項和為 sss 時，最後一項模 kkk 為 jjj 時的最大長度。

當我們遍歷 nums 陣列中的每一個元素 x = nums[i] 時，可以透過 選或不選 x 來更新 f：

  • 如果我們要將 x 作為有效子序列的最新一項，那麼它的前一項 prev 必須滿足 (prev+x)(modk)=s(prev + x) \pmod k = s(prev+x)(modk)=s。
  • 透過移項，我們可以得到 prev(modk)=(s−x)(modk)prev \pmod k = (s - x) \pmod kprev(modk)=(s−x)(modk)，也就是可以從 f[(s−x) mod k]f[(s - x) \bmod k]f[(s−x)modk] 轉移過來。
  • 如果不選 x，則 f[i][x]=f[i−1][x]f[i][x] = f[i - 1][x]f[i][x]=f[i−1][x]。

得到狀態轉移方程：

f[i][x]={max⁡(f[i−1][x], f[i−1][(s−x) mod k]+1),if x=nums[i] mod kf[i−1][x],otherwisef[i][x] = \begin{cases} \max\left(f[i-1][x],\ f[i-1][(s-x)\bmod k] + 1\right), & \text{if } x = \mathrm{nums}[i] \bmod k \\ f[i-1][x], & \text{otherwise} \end{cases} f[i][x]={max(f[i−1][x], f[i−1][(s−x)modk]+1),f[i−1][x],​if x=nums[i]modkotherwise​

由於從 f[i−1]f[i - 1]f[i−1] 轉移到 f[i]f[i]f[i] 時，只依賴於 f[i−1]f[i - 1]f[i−1] 的狀態，可以用 滾動陣列(Rolling Array) 的方式壓縮維度，又因為轉移時只需要多考慮最後一項為 x = nums[i] 的情況，因此除了 f[i][x]f[i][x]f[i][x] 之外，其餘的值都和 f[i−1]f[i - 1]f[i−1] 相同。

在這種寫法中，就算沒有注意到以下性質，直接對 選或不選 兩種情況取最大值也能通過。

值得注意的是，由於子序列中一定是 x,y,⋯ ,x,yx, y, \cdots, x, yx,y,⋯,x,y 交錯，因此在 選或不選 nums[i]nums[i]nums[i] 的兩種情況中，選一定不會比不選差。這是因為如果在上一個 yyy 之後、當前的 xxx 之前，有出現其他的 x′x^{\prime}x′，則此時 f[x]f[x]f[x] 的值已經且最多是 f[y]+1f[y] + 1f[y]+1；如果沒有，則選當前的 xxx 一定能得到更優的結果，可以用反證法證明。因此選 xxx 一定不會比不選差。

複雜度分析

  • 時間複雜度：O(k×(k+n))O(k \times (k + n))O(k×(k+n))
  • 空間複雜度：O(k)O(k)O(k)

方法二：紀錄最後兩項

由於省略了許多相似之處的說明，建議先觀看完方法一的說明再閱讀此方法。

首先，觀察到一個有效的子序列，其元素的餘數序列必定呈現 y,x,y,x,⋯y, x, y, x, \cdotsy,x,y,x,⋯ 這樣的交錯模式。因此，只需要紀錄最後兩項的餘數即可。

令 f[i][y][x]f[i][y][x]f[i][y][x] 表示考慮前 iii 個數，最後兩項的餘數為 y,xy, xy,x 時的最大長度，則可以得到狀態轉移方程：

f[i][y][x]={max⁡(f[i−1][y][x], f[i−1][x][y]+1),if x=nums[i] mod kf[i−1][y][x],otherwisef[i][y][x] = \begin{cases} \max\left(f[i-1][y][x],\ f[i-1][x][y] + 1\right), & \text{if } x = \mathrm{nums}[i] \bmod k \\ f[i-1][y][x], & \text{otherwise} \end{cases} f[i][y][x]={max(f[i−1][y][x], f[i−1][x][y]+1),f[i−1][y][x],​if x=nums[i]modkotherwise​

同樣地，由於從 f[i−1]f[i - 1]f[i−1] 轉移到 f[i]f[i]f[i] 時，只依賴於 f[i−1]f[i - 1]f[i−1] 的狀態，可以用滾動的方式壓縮維度，又因為轉移時只需要多考慮最後一項為 x = nums[i] 的情況，因此除了 f[i][y][x]f[i][y][x]f[i][y][x] 之外，其餘的值都和 f[i−1]f[i - 1]f[i−1] 相同。

類似地，在 選或不選 nums[i]nums[i]nums[i] 的兩種情況中，選一定不會比不選差。

複雜度分析

  • 時間複雜度：O(k2+n⋅k)O(k^2 + n \cdot k)O(k2+n⋅k)
  • 空間複雜度：O(k2)O(k^2)O(k2)
